## 1. Operating System & Python Version

| Requirement | Specification |
|---|---|
| **OS** | Windows 10 or Windows 11, 64-bit |
| **Python** | 3.10.x (64-bit) — strongly recommended |
| **Architecture** | x86-64 only (camera DLL is 64-bit) |

> **Why Windows only?**
> The camera vendor's `scTDClib.dll` is a Windows-only binary. `PyDAQmx` also wraps NI-DAQmx, which runs only on Windows. The rest of the codebase is cross-platform Python.

> **Why Python 3.10 specifically?**
> The `scTDC` camera SDK wheel (if provided by the vendor) is typically compiled against a specific CPython ABI. Python 3.10 has the broadest compatibility with the NI-DAQmx ctypes interface and with `PyDAQmx`. Python 3.11+ may work but is untested against the current camera SDK version.

## 2. Setting Up a Virtual Environment

Working inside a virtual environment avoids conflicts with other Python projects on the same machine.

```bat
:: Step 1 — Create environment
python -m venv lsci_env

:: Step 2 — Activate
lsci_env\Scripts\activate

:: Step 3 — Upgrade pip
python -m pip install --upgrade pip

:: Step 4 — Install all pip-available packages
pip install -r requirements.txt

:: Step 5 — Verify
python -c "import numpy, cv2, customtkinter, pymodbus, serial, matplotlib, scipy, sklearn, PIL, pygame, tkdial, docx; print('All pip packages OK')"
```

At this point, `import scTDC` and `import PyDAQmx` will still fail — those require vendor installations described in Sections 3 and 4.

## 3. Camera SDK — scTDC

The `scTDC` module interfaces with the **scientific time-to-digital converter (TDC) camera** used in this platform (1920×1440 pixels, 12-bit depth). It is **not available on PyPI**.

### 3.1 What you need from the camera vendor

| File | Purpose |
|---|---|
| `scTDClib.dll` | Core camera driver/SDK DLL |
| `tdc_gpx3.ini` | Camera initialization configuration (hardware-specific) |
| `scTDC` Python wheel | Python wrapper for the DLL (e.g., `scTDC-x.y.z-cp310-win_amd64.whl`) |

Contact the camera vendor's support team and request the SDK package for your specific camera model and Python 3.10 / Windows 64-bit.

### 3.2 Installation

```bat
:: 1. Place DLL and ini file in the project root (same folder as new_GUI_sc_55.py)
::    The code does: os.add_dll_directory(script_dir)
::    This must be the SAME directory as the .py file.

:: 2. Install the Python wheel (filename will vary)
pip install path\to\scTDC-x.y.z-cp310-win_amd64.whl

:: 3. Verify
python -c "import scTDC; print('scTDC OK')"
```

### 3.3 Camera initialization file — tdc_gpx3.ini

The `tdc_gpx3.ini` file contains hardware-specific parameters (timing resolution, trigger settings, etc.) that are set by the vendor or lab technician. **Do not edit this file** unless directed by the camera vendor. The application loads it with:

```python
camera = scTDC.Camera(inifilepath="tdc_gpx3.ini", autoinit=False, lib=lib)
```

The ini file path is **relative to the script directory**. If the camera fails to initialize, check that `tdc_gpx3.ini` is present in the same folder as `new_GUI_sc_55.py`.

### 3.4 DLL loading

The application explicitly adds its own directory to the Windows DLL search path:

```python
script_dir = os.path.dirname(os.path.abspath(__file__))
os.add_dll_directory(script_dir)
```

This means `scTDClib.dll` (and any DLLs it depends on, such as FTDI or USB3 driver DLLs that the vendor may include) must all reside in the **same directory** as the Python script.

## 4. NI-DAQmx Runtime & PyDAQmx

The forepaw electrical stimulator uses a **National Instruments DAQ card** via the `PyDAQmx` Python wrapper. This requires the NI-DAQmx runtime to be installed first.

### 4.1 Install NI-DAQmx Runtime

1. Download NI-DAQmx from National Instruments (free):
   [https://www.ni.com/en/support/downloads/drivers/download.ni-daq-mx.html](https://www.ni.com/en/support/downloads/drivers/download.ni-daq-mx.html)
2. Minimum version: **NI-DAQmx 19.0**. Recommended: latest stable release.
3. Run the installer and **restart Windows** after installation.
4. Open **NI MAX** (Measurement & Automation Explorer) to verify the DAQ card is recognized.

### 4.2 Verify device name

The code hardcodes the device name `"Dev1"` in the NiDAQ channel configuration:

```python
mdaq.CreateDOChan("Dev1/port0/line0", "", PyDAQmx.DAQmx_Val_ChanForAllLines)
```

Open **NI MAX → Devices and Interfaces** and confirm your DAQ card is listed as `Dev1`. If it shows a different name (e.g., `Dev2`), update the string in `new_GUI_sc_55.py` at line ~1913 and ~5574.

### 4.3 Install PyDAQmx Python wrapper

```bat
:: Only after NI-DAQmx runtime is installed:
pip install PyDAQmx>=1.4.0

:: Verify
python -c "import PyDAQmx; print('PyDAQmx OK')"
```

### 4.4 Physical connection

| Signal | DAQ Card Terminal | Description |
|---|---|---|
| Digital Output | `Dev1/port0/line0` | Pulse output to stimulator input |
| Ground | DAQ GND | Common ground with stimulator |

Set the stimulator hardware input mode to **DIRECT DIGITAL INPUT** when using pulse duration control from this software (the stimulator's own safety dialog reminds you of this at startup).

## 5. Serial / USB Driver Notes

The platform communicates with three categories of serial devices. Most USB-serial adapters require a driver that Windows may not install automatically.

### 5.1 Common USB-to-Serial chipsets and drivers

| Chipset | Devices | Driver |
|---|---|---|
| **CP2102 / CP2104** | Relay/Galvo controller, Laser Arduino | Silicon Labs CP210x: [silabs.com/developers/usb-to-uart-bridge-vcp-drivers](https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers) |
| **FT232R / FT232H** | Relay/Galvo alternative | FTDI VCP drivers: [ftdichip.com/drivers/vcp-drivers](https://ftdichip.com/drivers/vcp-drivers/) |
| **CH340 / CH341** | XYZ stereotaxic stage | CH340 driver: [wch.cn](http://www.wch.cn/download/CH341SER_EXE.html) |
| **Prolific PL2303** | XYZ stage alternative | Prolific: [prolific.com.tw](https://www.prolific.com.tw/US/ShowProduct.aspx?p_id=225&pcid=41) |

Install the relevant driver(s) for your hardware, then verify the COM port appears in **Device Manager → Ports (COM & LPT)**.

### 5.2 COM port auto-detection

At startup, the software scans all COM ports and matches by USB descriptor keywords. Check the console output for confirmation:

```
[PORT DETECTION] Available COM ports: ['COM3', 'COM4', 'COM7']
[PORT DETECTION] RELAY_GALVO found on COM4
[PORT DETECTION] XYZ_STAGE found on COM3
[PORT DETECTION] LASER found on COM7
```

If detection fails, the software falls back to: COM4 (Relay/Galvo), COM3 (XYZ), COM7 (Laser). You can override these defaults in the `auto_detect_ports()` function in the source file if your hardware uses fixed COM port numbers.

### 5.3 Baud rates in use

| Device | Baud Rate | Protocol |
|---|---|---|
| Relay / Galvo / Z-axis | 9600 | Modbus RTU, 8N1 |
| XYZ Stereotaxic Stage | 9600 | Modbus RTU |
| Laser Current Monitor | 19200 | ASCII serial, readline |

## 6. Verifying the Full Install

Run this checklist script from the project directory (with virtual environment active):

```python
# save as check_env.py and run: python check_env.py

import sys
print(f"Python: {sys.version}")
print(f"Platform: {sys.platform}")
print()

packages = [
    ("numpy",          "numpy"),
    ("opencv-python",  "cv2"),
    ("Pillow",         "PIL"),
    ("scipy",          "scipy"),
    ("scikit-learn",   "sklearn"),
    ("matplotlib",     "matplotlib"),
    ("customtkinter",  "customtkinter"),
    ("tkdial",         "tkdial"),
    ("pymodbus",       "pymodbus"),
    ("pyserial",       "serial"),
    ("python-docx",    "docx"),
    ("pygame",         "pygame"),
    ("PyDAQmx",        "PyDAQmx"),      # Needs NI-DAQmx runtime
    ("scTDC",          "scTDC"),        # Needs vendor SDK
]

all_ok = True
for pip_name, import_name in packages:
    try:
        mod = __import__(import_name)
        ver = getattr(mod, "__version__", "unknown")
        print(f"  OK   {pip_name:<20} (v{ver})")
    except ImportError as e:
        print(f"  FAIL {pip_name:<20} --> {e}")
        all_ok = False

print()
if all_ok:
    print("All packages imported successfully.")
else:
    print("Some packages failed. See ENVIRONMENT.md for setup instructions.")
```

## 7. Known Issues & Fixes

### Issue: `OSError: [WinError 126] The specified module could not be found` on import of scTDC

**Cause:** `scTDClib.dll` (or a dependency DLL it loads) is not in the DLL search path.

**Fix:** Ensure `scTDClib.dll` and all accompanying vendor DLLs are in the **exact same directory** as `new_GUI_sc_55.py`. Do not place them in `C:\Windows\System32` or on `PATH` — the code uses `os.add_dll_directory()` which only applies to the specified folder.

---

### Issue: `DAQmxFailed` / `DAQError -200220` when starting stimulation

**Cause:** NI-DAQmx runtime not installed, or device not found as `Dev1`.

**Fix:**
1. Install NI-DAQmx runtime (Section 4.1).
2. Open NI MAX and confirm the device name matches `Dev1`.
3. If device name differs, edit the `CreateDOChan` call in the source.

---

### Issue: `ModbusSerialClient` connects but Galvo/Relay does not respond

**Cause:** Incorrect slave ID or register address, or RS-485 termination missing.

**Fix:**
- Galvo and Relay use slave ID 2, address 0.
- XYZ stage uses slave ID 1, address 1.
- Ensure RS-485 bus termination resistor (120Ω) is in place on long cable runs.
- Verify the baud rate of the controller firmware matches 9600 baud.

---

### Issue: `serial.SerialException: could not open port` for Laser Monitor

**Cause:** Laser Arduino not recognized, or driver not installed.

**Fix:** Install the CP2102 or CH340 driver as appropriate (Section 5.1). Confirm the COM port in Device Manager. The software falls back to COM7 — override `laser_port_var` in the GUI's port selector if needed.

---

### Issue: `ImportError: DLL load failed while importing cv2`

**Cause:** OpenCV requires the Microsoft Visual C++ 2015–2022 Redistributable.

**Fix:** Download and install VC++ Redistributable (x64):
[https://aka.ms/vs/17/release/vc_redist.x64.exe](https://aka.ms/vs/17/release/vc_redist.x64.exe)

---

### Issue: `ModuleNotFoundError: No module named 'tkinter'`

**Cause:** Only relevant on Linux/macOS. tkinter is not bundled by default.

**Fix:**
```bash
sudo apt install python3-tk   # Ubuntu/Debian
brew install python-tk        # macOS with Homebrew
```

On Windows with the official Python installer, tkinter is always included.

## 8. Dependency Version Rationale

This section explains why specific minimum versions are required — relevant if you need to update or audit the dependency tree.

| Package | Min Version | Reason |
|---|---|---|
| `numpy` | 1.24 | `np.stack` signature stabilized; `np.uint16` dtype handling improved |
| `Pillow` | 10.0 | `Image.Resampling.LANCZOS` API (replaces deprecated `Image.LANCZOS`). Used directly in source code |
| `opencv-python` | 4.8 | `cv2.imwrite` 16-bit PNG stability on Windows |
| `pymodbus` | 3.5 | `ModbusSerialClient` constructor signature changed in v3.x vs v2.x (breaking change). The v3 API is used in the source |
| `customtkinter` | 5.2 | `CTkButton`, `CTkStringVar` stable API. Earlier versions had different constructor signatures |
| `matplotlib` | 3.7 | `FigureCanvasTkAgg`, `plt.imsave` with `vmin/vmax` args |
| `scikit-learn` | 1.3 | `LinearRegression.fit()` — no breaking changes, but 1.3+ recommended for stability |
| `python-docx` | 1.0 | `Document.add_table()` row/cell API. Pre-1.0 (`python-docx 0.8.x`) had a different namespace import |
| `pygame` | 2.5 | `pygame.mixer` initialization on Python 3.10 |
| `PyDAQmx` | 1.4 | ctypes interface matches NI-DAQmx 19.x+ C API |
